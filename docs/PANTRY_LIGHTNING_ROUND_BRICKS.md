# Pantry Lightning Round Bricks

Date: 2026-05-15
Status: Active setup plan after `a476ba2 Add pantry lightning pass controls`
Scope: Pantheon web backend, Supabase pantry governance, parser-facing `products`, identity aliases, and conservative USDA training packs.

## ELI5

Pantheon is about to start learning foods like loading skill disks.

The first disk is not "throw 1,000 foods into the database." The first disk is: prove the machine can safely add 10 to 25 boring trusted USDA foods, track where they came from, prove the parser still behaves, then repeat.

Once that works three times in a row, we can move faster. Until then, the rule is simple:

- trusted USDA rows can be added automatically when the risk engine says they are boring
- branded, restaurant, alcohol, supplement, recipe, Chipotle, and weird rows become review cards
- wrong-but-confident is worse than slow

## Operating Rules For This Round

Codex may continue without interruption for:

- docs, scripts, tests, pack files, read-only audits, dry-runs, and local reports
- committing and pushing Green work after checks pass
- applying only trusted USDA auto-approved rows when:
  - migration `021` is verified
  - `scripts/verify-pantry-governance.ts` passes
  - the artifact was generated by the current risk engine
  - insert count is at or below `--max-insert=25`
  - the manual report scan finds no suspicious auto-approved row

Codex must stop before:

- Supabase auth workarounds
- destructive changes or rollbacks
- Vercel production setting changes
- branded/OFF/Chipotle/restaurant/alcohol/supplement/recipe/LLM-estimated live writes
- raising the live insert cap above 25
- any novel bad-match class
- any auto-approved row that looks semantically wrong

## Why This Order

The first bottleneck is not imagination; it is trust.

Once the schema gate and first live apply work, every later pack becomes easier. Once three batches are clean, the risk engine has evidence. Once the approval ledger exists, branded and restaurant foods can move without turning Luke into a database admin.

This order keeps Pantheon aimed at the elite target:

- fast because high-frequency foods become local identity rows
- accurate because unsafe rows stay review-gated
- compounding because every pack leaves behind products, aliases, units, rejections, and tests

## Definition Of Done For The Round

This lightning round is successful when:

- governance schema is verified
- at least three safe USDA batches are applied
- pantry product count grows with provenance
- unit probes prove natural quantity handling
- review packets exist for Mexican/Chipotle and BBQ
- no branded/restaurant rows are written without approval
- parser tests stay green
- we have evidence for whether to raise batch cap to 50
